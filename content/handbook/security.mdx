Every new hire acknowledges this on day one (see [onboarding](/content/handbook/onboarding.mdx)). It is also the first reference when an [incident](/content/processes/incident-response.mdx) is declared.

## Accounts

- Hardware keys for anything production-adjacent
- No shared logins — ask Grove to add a person to the [directory](/content/directory.mdx) instead
- Rotate API tokens every `90 days`

## If something looks wrong

<Callout variant="warning" title="Report first, fix second">Page the on-call security lead before you touch anything. The [incident runbook](/content/processes/incident-response.mdx) takes over from there.</Callout>
